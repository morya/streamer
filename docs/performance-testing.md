# 性能测试与优化指南

本文档提供屏幕流媒体工具的性能测试方案和优化建议。

## 测试环境要求

### 硬件要求
- **CPU**: Intel i5 8代或同等性能以上
- **GPU**: NVIDIA GTX 1060或以上（支持NVENC）
- **内存**: 8GB RAM 或以上
- **存储**: SSD 推荐

### 软件要求
- **操作系统**: Windows 10/11 64位
- **Python**: 3.11.9
- **FFmpeg**: 6.1.1+（支持NVENC、SRT、RTMP）
- **驱动程序**: 最新NVIDIA显卡驱动

## 性能测试方案

### 1. FPS稳定性测试

#### 测试目标
验证在不同分辨率下能否稳定维持60 FPS。

#### 测试步骤
```bash
# 安装依赖
pip install -r requirements.txt

# 运行性能测试脚本
python tests/performance_fps.py
```

#### 测试分辨率
- 1080p (1920x1080)
- 1440p (2560x1440)
- 4K (3840x2160)

#### 合格标准
- 平均FPS ≥ 55
- FPS波动 ≤ 10%
- 无卡顿现象

### 2. 端到端延迟测试

#### 测试目标
测量从屏幕捕获到网络传输的全程延迟。

#### 测试方法
1. 在屏幕上显示实时计时器
2. 捕获屏幕并推流到本地服务器
3. 在接收端测量显示时间差

#### 测试工具
- **发送端**: 本应用程序
- **接收端**: VLC播放器或FFplay
- **测量工具**: 高速相机或软件时间戳

#### 合格标准
- 1080p分辨率下延迟 ≤ 200ms
- 4K分辨率下延迟 ≤ 300ms

### 3. 网络中断恢复测试

#### 测试目标
验证网络中断后的自动重连能力。

#### 测试步骤
1. 开始推流
2. 模拟网络中断（断开网线/禁用网络适配器）
3. 等待10秒后恢复网络
4. 观察恢复时间和数据完整性

#### 合格标准
- 网络恢复后30秒内自动重连
- 重连后视频流无花屏或卡顿
- 配置信息不丢失

## 性能优化建议

### 1. 捕获优化

#### DXcam配置
```python
# 使用DXcam高性能模式
stream_params = {
    "backend": "dxcam",
    "region": region,
    "output_color": "RGB",  # 避免颜色转换
    "max_buffer_len": 2,    # 最小化缓冲
}
```

#### 帧率控制
- 使用`target_fps`参数精确控制帧率
- 避免使用`sleep()`进行帧率控制
- 根据系统负载动态调整FPS

### 2. 编码优化

#### NVENC参数优化
```python
output_params = {
    "-vcodec": "h264_nvenc",
    "-preset": "fast",      # 平衡性能和质量
    "-tune": "ll",          # 低延迟模式
    "-rc": "cbr",           # 恒定码率
    "-g": 60,               # GOP大小匹配FPS
    "-bf": 2,               # B帧数量
    "-refs": 3,             # 参考帧数量
}
```

#### 码率自适应
- 根据网络条件动态调整码率
- 实现前向纠错（FEC）机制
- 添加带宽估计算法

### 3. 网络优化

#### SRT参数优化
```python
srt_params = {
    "mode": "caller",
    "latency": 200000,      # 200ms延迟
    "maxbw": 10000000,      # 10Mbps最大带宽
    "passphrase": "secret", # 加密传输
    "pbkeylen": 16,         # 加密密钥长度
}
```

#### 缓冲管理
- 实现自适应缓冲
- 添加丢包重传机制
- 支持前向纠错

### 4. 内存优化

#### 帧缓冲管理
- 使用固定大小的环形缓冲
- 及时释放不再使用的帧
- 避免不必要的内存拷贝

#### 资源清理
- 实现正确的析构函数
- 使用上下文管理器
- 添加资源泄漏检测

## 常见问题排查

### 1. FPS不稳定

#### 可能原因
- 系统资源不足
- DXcam初始化失败
- 编码器过载

#### 解决方案
1. 检查任务管理器中的资源使用情况
2. 降低捕获分辨率或FPS
3. 更新显卡驱动程序
4. 关闭不必要的后台程序

### 2. 高延迟

#### 可能原因
- 网络条件差
- 编码参数不合适
- 系统缓冲过大

#### 解决方案
1. 使用有线网络连接
2. 调整SRT延迟参数
3. 减少GOP大小
4. 使用低延迟编码预设

### 3. 花屏或卡顿

#### 可能原因
- 网络丢包
- 编码错误
- 帧丢失

#### 解决方案
1. 检查网络连接质量
2. 降低码率设置
3. 启用前向纠错
4. 使用更稳定的编码预设

### 4. 应用程序崩溃

#### 可能原因
- 内存泄漏
- 线程冲突
- 驱动程序问题

#### 解决方案
1. 检查应用程序日志
2. 更新所有依赖库
3. 运行内存泄漏检测工具
4. 简化配置参数

  push:
    branches: [ main ]
  pull_request:
    branches: [ main ]

    - uses: actions/checkout@v3

        python -m pip install --upgrade pip
        pip install -r requirements.txt
        pip install pytest pytest-benchmark
        
    - name: Run unit tests
      run: pytest tests/ -v
      
    - name: Run performance tests
      run: python tests/performance_suite.py
      
    - name: Generate test report
